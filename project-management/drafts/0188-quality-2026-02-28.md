## Top Recommendations

1. **Quality check scripts: Fix stale `orchestrator/` path references** — P1, Quick (<1 day)
   - Evidence: All 3 tools (pytest-cov, vulture, wily) fail when run via the scripts. `ORCHESTRATOR_PYTHONPATH` env var points to `/Users/maxwilliams/dev/octopoid/orchestrator` (non-existent). The guard script also fails silently, allowing duplicate drafts.
   - Action: Update `run-quality-checks.sh`, `guard.sh` to replace `orchestrator` with `octopoid` throughout. Confirm or update `ORCHESTRATOR_PYTHONPATH` in whatever sets the job env.
   - Effort: quick | Priority: P1 (blocker for this agent to function)

2. **scheduler.py: Decompose the 2,287-line unmaintainable core loop** — P1, Large (>3 days)
   - Evidence: 2,287 lines, MI=0.08 (grade C — effectively unmaintainable by radon), 17% coverage, contains D-complexity functions: `sweep_stale_resources` (CC=24), `check_and_update_finished_agents` (CC=18), `_run_agent_evaluation_loop` (CC=13). The scheduler is the central process and any change risks silent breakage.
   - Action: Split the scheduler into focused sub-modules: (a) `agent_lifecycle.py` for spawn/monitor/reap logic, (b) `queue_monitor.py` for health/lease checks, (c) keep `scheduler.py` as a thin orchestrator. Add integration tests for each newly extracted module.
   - Effort: large | Priority: P1

3. **9 failing integration tests: Fix behavioral regressions in outcome routing and lease clearing** — P2, Medium (1–3 days)
   - Evidence: 9 tests failing in the integration suite with 3 patterns:
     - **Outcome routing** (4 tests): `outcome=failure`, `outcome=unknown`, `outcome=needs_continuation` all end up in `requires-intervention` instead of `failed`/`needs_continuation`. Tests: `test_agent_failure_goes_to_failed`, `test_agent_crash_goes_to_failed`, `test_dead_pid_detected_and_task_moved_to_failed`, `test_needs_continuation`. The result_handler logs `WARNING: unknown outcome 'unknown', routing to requires-intervention` even when the outcome is `failure` — suggesting the stdout inference (`infer_result_from_stdout`) is returning `unknown` for all outcomes.
     - **Lease not cleared** (1 test): `test_expired_provisional_lease_is_cleared` — `lease_expires_at` remains set after `check_and_requeue_expired_leases()` runs. The log shows requeue happening (`Requeued expired lease`) but the field isn't being cleared.
     - **Worktree path mismatch** (2 tests): `rebase_on_base` step looks for `task_dir/worktree` but test setup creates the worktree at `tmp_path/worktree`. Tests: `test_active_lease_not_requeued`, `test_agent_failure_goes_to_failed`.
   - Action: Fix `infer_result_from_stdout` to correctly parse failure/needs_continuation outcomes; fix `check_and_requeue_expired_leases` to clear `lease_expires_at`; fix test worktree path setup or make `rebase_on_base` more flexible.
   - Effort: medium | Priority: P2

4. **tasks.py + steps.py: Add tests for core task manipulation paths** — P2, Medium (1–3 days)
   - Evidence: `tasks.py` is 14% covered (345/403 statements untested). Untested functions include the entire `create_task` body (lines 52–84), all of `request_intervention` (306–411), `fail_task` (448–527), and the complete project/breakdown flow (635–939). `steps.py` is 12% covered (333/377 statements). These are invoked on every task completion — regressions here affect all agent workflows.
   - Action: Add targeted unit tests for `fail_task`, `request_intervention`, `submit_completion`, and the top-5 step functions (`rebase_on_base`, `push_branch`, `merge_pr`, `create_pr`, `update_changelog`). Use mocked SDK to avoid server dependencies.
   - Effort: medium | Priority: P2

5. **queue_utils.py: Audit 22 re-exported symbols** — P3, Quick (<1 day)
   - Evidence: 22 symbols re-exported in `queue_utils.py` at lines 13, 21, 25, 29, 32 — all flagged by vulture at 90% confidence. These include `accept_completion`, `complete_task`, `hold_task`, `reject_task`, `reset_task`, `resume_task`, `retry_task`, `review_reject_task`, `submit_completion`, `unclaim_task`, `mark_needs_continuation`, `get_review_feedback`, `activate_project`, `approve_project_via_flow`, `get_project_status`, `send_to_breakdown`, `clear_task_marker`, `read_task_marker`, `write_task_marker`, `get_task_notes`, `save_task_notes`, `can_create_task`, `get_queue_status`.
   - Action: Search for usage of each symbol in `agents/`, `commands/`, `scripts/`, and any external callers. Remove symbols that are genuinely unused; keep those that are part of the public API.
   - Effort: quick | Priority: P3
